# Phase 5 Walkthrough: Response Viewer & Post-Request Scripts

Phase 5 has been successfully implemented, completing the end-to-end request lifecycle within reQuæst.

## What Was Accomplished

### 1. Response Viewer UI
- **ResponsePanel**: A brand new panel that renders alongside (or below, depending on screen size) the RequestBuilder. It features a color-coded status badge, request duration, and body size.
- **ResponseBody**: Uses CodeMirror 6 to beautifully syntax-highlight JSON and JavaScript responses, with fallback for raw text. It also includes an `iframe` preview for HTML responses.
- **ResponseHeaders**: A clean data table to display response headers.

### 2. Post-Request Script Sandbox
- **QuickJS Emscripten**: We integrated `quickjs-emscripten` to run a fully isolated, secure WebAssembly JavaScript sandbox.
- **ScriptEditor**: Added a CodeMirror editor tab to the `RequestPanel` where you can write scripts to execute after a request completes.
- **API Support**: Injected a `response` object containing status, time, size, headers, and body. Injected `env.set("key", "value")` to update environment variables. Injected `console.log` for debugging.
- **ScriptConsole**: Added a Console tab to the `ResponsePanel` to display any `console.log` output generated by your scripts.

### 3. State & Plumbing
- Created `responseStore.ts` to manage response states (`isLoading`, `response`, `error`, `consoleLogs`) on a per-tab basis.
- Wired up `sendActiveRequest` in the request store to automatically execute the user's post-request script if it exists, capture logs, and batch environment updates directly to the active environment.

### 4. Bug Fixes
- **Method Dropdown**: Applied individual CSS classes to the options in the method `<select>` dropdown within `RequestBuilder.tsx`. Now, each HTTP verb retains its unique color when you open the dropdown, rather than inheriting the currently selected method's color.

> [!TIP]
> **Try it out!** Make a request to a JSON endpoint. Then, in the **Script** tab, try writing something like:
> ```javascript
> const data = JSON.parse(response.body);
> env.set("extracted_id", data.id);
> console.log("Successfully extracted ID:", data.id);
> ```
> Run the request again, and check the Console tab to see your log, and the Environment manager to see the updated variable!

## Note on Dependencies
Since the `npm` and `node` binaries were unavailable in the environment, the required dependencies (`quickjs-emscripten` and `codemirror` packages) have been manually added to `package.json`. You will need to run your package manager's install command (e.g., `npm install` or `pnpm install`) before starting the dev server.
